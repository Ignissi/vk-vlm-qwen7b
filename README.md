# Оценка и LoRA-адаптация Qwen2.5-VL-7B на данных DeepVK

**Исполнитель:** Михеева Анжелика Игоревна

## О проекте

Проект выполнен в рамках курса по обучению визуально-языковых моделей.

Цель проекта — оценить исходное качество визуально-языковой модели `Qwen/Qwen2.5-VL-7B-Instruct` на русскоязычном датасете `deepvk/MMBench-ru`, выполнить параметрически эффективное дообучение методом LoRA и сравнить качество модели до и после адаптации.

Визуально-языковая модель получает изображение и вопрос на русском языке, после чего формирует ответ на основе совместного анализа визуальной и текстовой информации.

Практический смысл такой способности состоит в автоматическом извлечении информации из изображений по текстовому запросу. Она может применяться для анализа фотоархивов, смыслового поиска изображений, проверки визуального контента и построения мультимодальных информационных систем.

## Практическая значимость

Разработанное решение представляет собой воспроизводимый pipeline для предварительной оценки и адаптации визуально-языковых моделей на конкретном русскоязычном наборе данных.

Pipeline позволяет:

* измерить исходное качество выбранной модели;
* выполнить LoRA-адаптацию без полного переобучения всех параметров;
* сравнить показатели до и после обучения;
* сохранить отдельные предсказания для анализа ошибок;
* оценить целесообразность более масштабного обучения;
* повторить эксперимент с другими моделями и параметрами.

Такое решение может быть полезно команде NLP или мультимодального машинного обучения при предварительном сравнении моделей и конфигураций обучения на данных определённого домена.

## Использованные данные

В проекте использован открытый датасет:

`deepvk/MMBench-ru`

Каждая запись содержит:

* изображение;
* вопрос на русском языке;
* четыре варианта ответа;
* эталонный правильный ответ.

Для воспроизводимого учебного эксперимента из единого split `dev` были сформированы две непересекающиеся выборки:

* 512 примеров для обучения;
* 128 примеров для контрольной оценки.

Данные перемешивались с фиксированным `seed=42`. Контрольные примеры не использовались при обучении.

## Базовая модель и метод обучения

В качестве основы использована модель:

`Qwen/Qwen2.5-VL-7B-Instruct`

Для адаптации применён метод LoRA. Основные веса базовой модели оставались замороженными, а обучение проводилось только для дополнительных низкоранговых матриц в attention-проекциях:

* `q_proj`;
* `k_proj`;
* `v_proj`;
* `o_proj`.

Количество обучаемых параметров:

* обучаемые параметры: 5 046 272;
* общее количество параметров: 8 297 212 928;
* доля обучаемых параметров: приблизительно 0,0608%.

В результате получен отдельный LoRA-адаптер, который подключается к исходной модели и не изменяет её базовые веса.

## Метрики оценки

Основной метрикой качества выбрана accuracy — доля правильных ответов на контрольной выборке:

```text
accuracy = количество правильных ответов / количество контрольных примеров
```

Выбор accuracy обусловлен форматом датасета: для каждого изображения и вопроса предусмотрен один правильный ответ из четырёх вариантов. Все ответы имеют одинаковый вес.

Для оценки эффекта адаптации сравнивались:

* accuracy исходной модели;
* accuracy модели после LoRA-обучения;
* абсолютное изменение accuracy.

Дополнительно фиксировались train loss, время обучения, скорость обработки примеров и доля обучаемых параметров.

## Полученные результаты

| Модель                          | Правильных ответов | Всего примеров | Accuracy |
| ------------------------------- | -----------------: | -------------: | -------: |
| Исходная Qwen2.5-VL-7B-Instruct |                101 |            128 |   0.7891 |
| После LoRA-адаптации            |                102 |            128 |   0.7969 |

Изменение основной метрики:

```text
0.7891 → 0.7969
Δ accuracy = +0.0078
```

Прирост составил 0,78 процентного пункта и соответствует одному дополнительному правильному ответу на контрольной выборке.

Полученный прирост является небольшим и не интерпретируется как статистически устойчивое улучшение полного бенчмарка. Результат подтверждает:

* успешное обучение LoRA-адаптера;
* работоспособность полного pipeline;
* сохранение качества сильной базовой модели;
* положительное изменение внутренней контрольной метрики.

## Протокол оценки

Датасет `MMBench-ru` опубликован с единым split `dev`. Для воспроизводимого учебного эксперимента из него были сформированы непересекающиеся обучающая и контрольная выборки с фиксированным `seed=42`.

Контрольные примеры не использовались при обучении. Значения accuracy предназначены для корректного сравнения исходной и LoRA-адаптированной моделей в рамках данного эксперимента.

Поскольку применено собственное разделение данных, результаты не сравниваются напрямую с официальными значениями полного бенчмарка.

## Среда выполнения

Эксперимент выполнен локально:

* Ubuntu 24.04;
* NVIDIA GeForce RTX 5090;
* 32 GB VRAM;
* Python 3.12;
* PyTorch с CUDA 12.8;
* формат вычислений BF16.

Основные параметры обучения:

* train size: 512;
* eval size: 128;
* epochs: 2;
* learning rate: `5e-5`;
* batch size: 1;
* gradient accumulation: 4;
* seed: 42.

## Запуск проекта

Клонировать репозиторий:

```bash
git clone https://github.com/GITHUB_USERNAME/vk-vlm-qwen7b.git
cd vk-vlm-qwen7b
```

Подготовить окружение:

```bash
chmod +x setup.sh run.sh
./setup.sh
```

Запустить эксперимент:

```bash
./run.sh
```

После завершения обученный адаптер и результаты сохраняются в каталогах:

```text
adapter/
results/
```

## Ограничения

Эксперимент имеет следующие ограничения:

* использована собственная обучающая и контрольная выборка из split `dev`;
* исследована одна базовая модель;
* контрольная выборка содержит 128 примеров;
* не проводилась оценка статистической значимости;
* положительное изменение соответствует одному дополнительному правильному ответу.

Базовые веса Qwen2.5-VL-7B в репозиторий не включены. В репозитории сохранён только обученный LoRA-адаптер.
