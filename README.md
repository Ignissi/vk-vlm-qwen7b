# Оценка и LoRA-адаптация Qwen2.5-VL-7B на данных DeepVK

**Исполнитель:** Михеева Анжелика Игоревна

## Суть проекта

Проект посвящён обучению и оценке визуально-языковой модели, способной отвечать на вопросы о содержимом изображений на русском языке.

На вход модели передаются:

1. изображение;
2. вопрос на русском языке;
3. от двух до четырёх заполненных вариантов ответа, хранящихся в полях `A–D`.

Модель должна проанализировать изображение, понять текст вопроса и вернуть букву правильного варианта.

Буква является форматом автоматической проверки. Фактически оценивается способность модели:

* распознавать объекты и свойства изображения;
* понимать действия и отношения между объектами;
* определять пространственное расположение;
* извлекать информацию из диаграмм, таблиц и схем;
* связывать визуальную информацию с вопросом на русском языке.

## Цель проекта

Оценить исходное качество визуально-языковой модели `Qwen/Qwen2.5-VL-7B-Instruct` на русскоязычном датасете `deepvk/MMBench-ru`, выполнить параметрически эффективное дообучение методом LoRA и определить, как адаптация влияет на точность ответов модели.

## Что было сделано

В рамках проекта выполнена полная экспериментальная цепочка:

1. Загружен открытый датасет `deepvk/MMBench-ru`.
2. Сформированы непересекающиеся обучающая и контрольная выборки.
3. Измерена исходная точность `Qwen2.5-VL-7B-Instruct`.
4. Обучен отдельный LoRA-адаптер.
5. Модель повторно оценена на той же контрольной выборке.
6. Сохранены обученные веса, метрики и подробные предсказания.
7. Проведено сравнение результатов до и после адаптации.

## Практическая значимость

Проект создаёт воспроизводимую процедуру предварительной оценки визуально-языковых моделей на конкретном наборе русскоязычных данных.

Такое решение может использоваться командой NLP или мультимодального машинного обучения для того, чтобы:

* проверить, насколько хорошо модель понимает изображения и русскоязычные вопросы;
* сравнить исходную и адаптированную версии модели;
* определить, даёт ли дополнительное обучение измеримый эффект;
* изучить конкретные ошибки модели;
* оценить целесообразность более масштабного и дорогостоящего обучения;
* повторить эксперимент с другой моделью или другими параметрами.

Таким образом, результатом проекта является не только обученный адаптер, но и готовый pipeline для оценки производительности визуально-языковой модели на данных DeepVK.

## Использованные данные

В проекте использован открытый датасет:

`deepvk/MMBench-ru`

Датасет содержит 3910 русскоязычных мультимодальных заданий.

Каждая запись включает:

* `image` — изображение;
* `question` — вопрос;
* `hint` — дополнительный контекст, если он предусмотрен;
* `A`, `B`, `C`, `D` — поля возможных вариантов ответа;
* `answer` — букву правильного ответа.

В отдельных заданиях заполнены не все четыре поля вариантов. Фактическое количество доступных ответов может составлять два, три или четыре. При формировании запроса используются только заполненные варианты.

Для эксперимента были выделены:

* 512 примеров для обучения;
* 128 других примеров для контрольной оценки.

Данные перемешивались с фиксированным `seed=42`. Контрольные примеры не использовались при обучении.

## Базовая модель

В качестве основы выбрана:

`Qwen/Qwen2.5-VL-7B-Instruct`

Это предварительно обученная визуально-языковая модель, способная совместно обрабатывать изображение и текст.

Проект не обучает 7B-модель с нуля. Вместо этого к ней добавляется обучаемый LoRA-адаптер, который специализирует модель для выбранного формата русскоязычных мультимодальных заданий.

## Метод обучения

Для адаптации применён метод LoRA.

Основные веса Qwen2.5-VL-7B оставались замороженными. Обучались дополнительные низкоранговые матрицы в attention-проекциях:

* `q_proj`;
* `k_proj`;
* `v_proj`;
* `o_proj`.

Количество параметров:

* обучаемые параметры: 5 046 272;
* общее количество параметров: 8 297 212 928;
* доля обучаемых параметров: приблизительно 0,0608%.

Обученный адаптер сохраняется отдельно в файле:

`adapter/adapter_model.safetensors`

## Метрика оценки

Основной метрикой выбрана accuracy:

```text
accuracy = количество правильных ответов / количество контрольных примеров
```

Модель получает изображение, вопрос и доступные варианты, после чего должна вернуть одну букву правильного ответа.

Предсказание считается правильным, если сгенерированная буква совпадает с полем `answer`.

Accuracy подходит для этой постановки, потому что:

* для каждого задания существует один эталонный ответ;
* варианты ответа равноправны;
* все ошибки имеют одинаковый вес;
* требуется прямое сравнение исходной и адаптированной моделей.

Дополнительно сохраняются:

* train loss;
* время обучения;
* скорость обучения;
* доля обучаемых параметров;
* предсказания модели для каждого контрольного примера.

## Полученные результаты

| Модель                          | Правильных ответов | Всего примеров | Accuracy |
| ------------------------------- | -----------------: | -------------: | -------: |
| Исходная Qwen2.5-VL-7B-Instruct |                101 |            128 |   0.7891 |
| После LoRA-адаптации            |                102 |            128 |   0.7969 |

Изменение метрики:

```text
0.7891 → 0.7969
Δ accuracy = +0.0078
```

Прирост составил 0,78 процентного пункта и соответствует одному дополнительному правильному ответу.

Полученный прирост является небольшим, поэтому он не рассматривается как доказательство значительного общего улучшения модели.

Эксперимент при этом подтвердил, что:

* LoRA-адаптер был успешно обучен;
* сильное исходное качество модели сохранилось;
* на контрольной выборке получено положительное изменение accuracy;
* вся цепочка обучения и оценки работает воспроизводимо.

## Протокол оценки

`MMBench-ru` опубликован с единым split `dev`.

Для учебного эксперимента из него были сформированы непересекающиеся обучающая и контрольная выборки с фиксированным `seed=42`.

Контрольные примеры не использовались во время обучения.

Полученные значения accuracy предназначены для сравнения исходной и LoRA-адаптированной моделей внутри данного эксперимента. Поскольку применено собственное разделение данных, результат не позиционируется как официальный показатель полного MMBench-ru.

## Среда выполнения

Эксперимент выполнен локально:

* Ubuntu 24.04;
* NVIDIA GeForce RTX 5090;
* 32 GB VRAM;
* Python 3.12;
* PyTorch с CUDA 12.8;
* BF16.

Параметры обучения:

* train size: 512;
* eval size: 128;
* epochs: 2;
* learning rate: `5e-5`;
* batch size: 1;
* gradient accumulation: 4;
* seed: 42.

## Ограничения

Эксперимент имеет следующие ограничения:

* использовано собственное разделение единого split `dev`;
* исследована одна базовая модель;
* контрольная выборка содержит 128 примеров;
* не проводилась оценка статистической значимости;
* положительное изменение соответствует одному дополнительному правильному ответу.

Базовые веса Qwen2.5-VL-7B в репозиторий не включены. Для запуска они автоматически загружаются из открытого источника. В репозитории хранится только обученный LoRA-адаптер.
